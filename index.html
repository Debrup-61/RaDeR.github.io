Abstract

We propose RaDeR, a set of reasoning-based dense retrieval models trained with data derived from mathematical problem solving using large language models (LLMs). Our method lever- ages retrieval-augmented reasoning trajectories of an LLM and self-reflective relevance evalua- tion, enabling the creation of both diverse and hard-negative samples for reasoning-intensive relevance. RaDeR retrievers, trained for math- ematical reasoning, effectively generalize to diverse reasoning tasks in the BRIGHT and RAR-b benchmarks, consistently outperform- ing strong baselines in overall performance. Notably, RaDeR achieves significantly higher performance than baselines on the Math and Coding splits. In addition, RaDeR presents the first dense retriever that outperforms BM25 when queries are Chain-of-Thought reason- ing steps, underscoring the critical role of reasoning-based retrieval to augment reason- ing language models. Furthermore, RaDeR achieves comparable or superior performance while using only 2.5% of the training data used by the concurrent work REASONIR, highlight- ing the quality of our synthesized training data.

Motivation

❓ Types of Queries used for Retrieval Training

❓ MCTS CoT Reasoning Query

We use the CoT reasoning steps (partial solution) up to the query node directly from the MCTS as the query.

❓ LLM Reasoning Query

We prompt a LLM to generate a reasoning query based on the math question M , the reasoning steps up to the query node (partial solution), and the theorem node from MCTS.

❓ Lexical Query

We generate queries that have high term similarity with their respective relevant theorems, by prompting a LLM using only the theorem node from the MCTS.

🔥 RaDeR achieves State-of-the-Art!

RaDeR achieves state-of-the-art performance on the BRIGHT benchmark, reaching an nDCG@10 of 39.2 with the RaDeR gte-Qwen2-7B model trained on MATH queries (CoT + LLMq + lexical).

In this setting, we rerank the top 100 documents from a RaDeR + BM25 hybrid approach, using Qwen-2.5-32B-Instruct following concurrent work ReasonIR.

🔍 Success Example: RaDeR vs Other Retrievers

🚀 RaDeR successfully performs nuanced reasoning to accurately retrieve the gold theorem, while a strong baseline (Qwen2) fails to do so. This illustrates RaDeR's ability to bridge gaps between reasoning depth and retrieval performance.

🧠 Performance on Coding Splits

🚀 RaDeR achieves the best performance on LeetCode, surpassing strong baselines by 8 points in the query=question setting. Results on Pony in the CoT reasoning setting also show notable improvements. These results are especially significant as they demonstrate that training our retrievers for mathematical reasoning yields substantial improvements in code retrieval—despite no code-specific training data.
